# 贡献指南

本项目使用统一的提交信息前缀，便于后续生成变更日志与版本管理。

## Commit 前缀规范

提交格式：

```text
<type>: <中文描述>
```

示例：

```text
feat: 新增记忆导入导出能力
fix: 修复流式消息在工具调用后被覆盖的问题
refactor: 拆分 model_runtime 降低单文件复杂度
chore: 调整构建脚本与开发工具配置
docs: 更新计划文档与使用说明
style: 优化配置窗口字段排版与间距
test: 增加流式输出缓冲逻辑测试
```

## type 说明

- `feat`: 新功能
- `fix`: 缺陷修复
- `refactor`: 重构（不改变外部功能）
- `chore`: 工程维护（依赖、脚本、配置等）
- `docs`: 文档改动
- `style`: 仅样式或格式调整（不改逻辑）
- `test`: 测试相关改动

## 约定建议

- 描述尽量具体，直接说明“改了什么”。
- 一次提交只做一件事，避免混合多个主题。
- 默认使用中文描述，便于团队沟通。
